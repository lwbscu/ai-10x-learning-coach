# Assessment Rubric

Use this reference when grading mastery checks and diagnosing learner answers.

## Diagnosis Labels

| Label | Meaning | Tutor Response |
|---|---|---|
| `correct` | The learner can explain the concept and apply it in context. | Advance or ask one transfer question. |
| `partially-correct` | The main idea is present but incomplete. | Fill the missing piece and ask a focused follow-up. |
| `misconception` | The learner's model points in the wrong direction. | Name the misconception, contrast with the correct model, retest. |
| `missing-boundary` | The learner knows when it works, not when it fails. | Add edge cases and a boundary question. |
| `application-gap` | The learner can define it but cannot use it. | Give a scenario and require a decision. |
| `term-confusion` | Two similar terms are mixed together. | Build an A/B comparison table and ask for a distinction. |

## Feedback Shape

Use this structure:

```text
Verdict: partially-correct
What you got right:
What is missing:
The precise confusion:
Corrected version:
Retest question:
```

## Question Recipes

Concept distinction:

```text
Explain the difference between A and B. Give one case where choosing A is wrong.
```

Application:

```text
Given this scenario, which concept/tool/process would you use and why?
```

Boundary:

```text
When does this idea stop working? Name one exception or failure mode.
```

Error diagnosis:

```text
Someone says: "...". What is wrong with this reasoning?
```

Teach-back:

```text
Explain this module to a smart beginner in 5 sentences. Avoid jargon unless you define it.
```
